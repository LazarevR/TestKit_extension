# TestKit — расширение для браузера

**TestKit** — инструмент для разработчиков и тестировщиков, который живёт в контекстном меню браузера. Вставляй тестовые данные в поля, определяй технологии на странице, проверяй HTTP-заголовки безопасности, анализируй мета-теги для SEO, тестируй XSS/SQL-уязвимости, очищай кэш сайта — не покидая вкладку.

> **Версия:** 1.0.2
> **Манифест:** v3
> **Браузеры:** Chrome, Yandex Browser, Firefox 121+, Edge

---

## Возможности

| Раздел | Описание |
|---|---|
| **Технологии сайта** | Сканирует страницу и показывает стек технологий с версиями |
| **HTTP-заголовки** | Анализирует заголовки безопасности (CSP, HSTS и др.), выявляет отсутствующие и проблемные |
| **Meta-теги (SEO)** | Показывает все мета-теги: Basic SEO, Open Graph, Twitter Card, технические |
| **Lorem Ipsum** | Вставка текста-заглушки на EN или RU, от 10 до 500 символов |
| **SQL-инъекции** | 17 готовых SQL-пейлоадов для тестирования полей ввода |
| **Скрипты (XSS)** | 18 XSS-векторов: классические, DOM, закодированные, полиглоты |
| **Номера телефонов** | Тестовые номера для 15 стран в 2–3 форматах каждая |
| **Имена** | Мужские и женские имена на русском, латинице и китайском |
| **Edge Cases** | Спецсимволы, Unicode, граничные числа, невалидные даты, пути и шаблоны |
| **E-mails** | Набор валидных и невалидных адресов |
| **Очистка кэша** | Сброс HTTP-кэша и жёсткая перезагрузка вкладки |
| **Очистка данных** | Одним кликом удаляет куки, localStorage, кэш и service worker текущего сайта |
| **Пользовательские меню** | Создавай свои группы и подгруппы с готовыми текстовыми сниппетами |
| **Хранилище** | Показывает объём данных, сохранённых текущим сайтом |
| **Импорт / Экспорт** | Бэкап и обмен пользовательскими меню в формате JSON |

## Установка

0. Скачай или клонируй этот репозиторий, а затем разархивируй его в любую удобную папку

### Chrome / Edge / Yandex Browser

1. Открой страницу расширений:
   - Chrome: `chrome://extensions`
   - Edge: `edge://extensions`
   - Yandex Browser: `browser://extensions`
2. Включи **Режим разработчика** (переключатель в правом верхнем углу)
3. Нажми **Загрузить распакованное**
4. Выбери папку `Extension` (ту, где лежит `manifest.json`)

Иконка шестерёнки TestKit появится на панели инструментов.

---

### Firefox

Firefox позволяет загружать расширение как временное дополнение (сбрасывается при перезапуске браузера):

1. Открой `about:debugging#/runtime/this-firefox`
2. Нажми **Загрузить временное дополнение…**
3. Перейди в папку `Extension` и выбери файл `manifest.json`

> Для постоянной установки в Firefox требуется подпись расширения.
> Подробнее: [Firefox Self-Distribution Guide](https://extensionworkshop.com/documentation/publish/self-distribution/)

## Как пользоваться

### Контекстное меню

Правый клик на любой странице открывает меню TestKit:

```
TestKit ▸
  ├── 🔍 Технологии сайта
  ├── 🔐 HTTP-заголовки
  ├── 🏷️ Meta-теги (SEO)
  ├── ─────────────────────────
  ├── 🔄 Очистка кэша и рефреш
  ├── 🗑️ Очистка данных сайта
  ├── ─────────────────────────
  ├── 📝 Lorem Ipsum ▸
  │     ├── EN — English ▸
  │     └── RU — Русский ▸
  ├── 💉 SQL Инъекции ▸ (17 штук)
  ├── 📜 Скрипты (XSS) ▸ (18 штук)
  ├── 📞 Номера телефонов ▸ (15 стран)
  ├── 👤 Имена ▸ (RU / EN / ZH)
  ├── 🧪 Edge Cases ▸
  │     ├── Спецсимволы ▸
  │     ├── Unicode ▸
  │     ├── Числа ▸
  │     ├── Даты ▸
  │     └── Пути и шаблоны ▸
  ├── 📧 E-mails ▸
  │     ├── Валидные ▸
  │     └── Невалидные ▸
  ├── [Твои пользовательские меню]
  ├── ─────────────────────────
  ├── ⚙️ Настройки TestKit
  ├── ─────────────────────────
  └── 💾 Хранилище: 0 Б
```

**Вставка текста:** кликни на любой пункт, пока курсор находится в `<input>`, `<textarea>` или `contenteditable` элементе — текст вставится в позицию курсора.

---

### Технологии сайта

Правый клик → **Технологии сайта** — открывает панель со стеком:

- Фреймворки и библиотеки (React, Vue, Angular, jQuery…)
- CMS и платформы (WordPress, Bitrix, Tilda…)
- Аналитика и трекеры
- Инструменты сборки и мониторинга

Панель использует локальную базу `technologies.json`, которую можно подключить к удалённому URL для автообновления (Настройки → База технологий).

---

### HTTP-заголовки

Правый клик → **HTTP-заголовки** — открывает панель анализа ответных заголовков сервера:

**Защитные заголовки** — проверяет наличие 9 ключевых заголовков безопасности:

| Заголовок | Важность | Защищает от |
|---|---|---|
| `Content-Security-Policy` | Критично | XSS-атак |
| `Strict-Transport-Security` | Критично | SSL-stripping, downgrade |
| `X-Frame-Options` | Важно | Clickjacking |
| `X-Content-Type-Options` | Рекомендовано | MIME-sniffing |
| `Referrer-Policy` | Рекомендовано | Утечки URL |
| `Permissions-Policy` | Рекомендовано | Злоупотребления API браузера |
| `Cross-Origin-Opener-Policy` | Опционально | XS-Leaks |
| `Cross-Origin-Embedder-Policy` | Опционально | Утечек через SharedArrayBuffer |
| `Cross-Origin-Resource-Policy` | Опционально | Несанкционированной загрузки ресурсов |

Отсутствующие заголовки отмечаются с уровнем критичности и готовой рекомендацией что и как добавить.

**Раскрытие информации** — предупреждает, если сервер возвращает `Server`, `X-Powered-By` и аналогичные заголовки, раскрывающие версии ПО.

**Технические детали:** для получения заголовков расширение делает отдельный `HEAD`-запрос к URL текущей страницы (с куками пользователя), не перезагружая вкладку.

---

### Meta-теги (SEO)

Правый клик → **Meta-теги (SEO)** — считывает все мета-теги со страницы и показывает их по группам:

- **Основные SEO** — `title`, `description`, `robots`, `canonical` и др.
- **Open Graph** — `og:title`, `og:description`, `og:image`, `og:url` и др.
- **Twitter / X** — `twitter:card`, `twitter:title`, `twitter:image` и др.
- **Техническое** — `viewport`, `charset`, `theme-color` и др.
- **Прочие теги** — все остальные мета-теги, найденные на странице

Важные отсутствующие теги (например, `og:image` или `twitter:card`) помечаются как **ОТСУТСТВУЕТ**.

---

### Навигация между панелями

Все три аналитические панели (Технологии, HTTP-заголовки, Meta-теги) связаны между собой. В верхней части каждой панели находятся кнопки переключения — можно перейти к другому разделу прямо из открытого окна без возврата к контекстному меню.

---

### Edge Cases

Данные, которые чаще всего ломают поля на сайтах:

| Группа | Примеры |
|---|---|
| **Спецсимволы** | `"`, `'`, `&`, `<>`, `\0`, `%00`, CRLF |
| **Unicode** | Эмодзи 😀🔥 (4 байта), RTL-текст مرحبا, нулевая ширина, строка 300 символов |
| **Числа** | `0`, `-1`, `99999999999999`, `1.000.000`, `NaN`, `Infinity` |
| **Даты** | `29.02.2023`, `31.11.2024`, `2038-01-19`, `9999-12-31` |
| **Пути и шаблоны** | `../../etc/passwd`, `{{7*7}}`, `${7*7}`, `<%= 7*7 %>` |

---

### Очистка данных сайта

Правый клик → **Очистка данных сайта** удаляет для **текущего origin**:
- Куки
- localStorage
- Кэш-хранилище
- Service workers
- HTTP-кэш

После этого вкладка перезагружается. Другие сайты не затрагиваются.

---

### Пользовательские меню

1. Кликни на иконку TestKit в панели инструментов (или открой **Настройки**)
2. Нажми **+ Добавить группу** — создаётся меню верхнего уровня
3. Внутри группы можно добавить:
   - **Текстовый пункт** — сниппет, который вставляется по клику
   - **Подгруппу** — вложенное подменю со своими пунктами
4. Кнопка редактирования у каждого пункта — изменить текст

**Экспорт:** Настройки → Импорт/Экспорт → Скачать `settings.json`
**Импорт:** Настройки → Импорт/Экспорт → Загрузить файл
*(Импорт заменяет все существующие пользовательские меню)*
